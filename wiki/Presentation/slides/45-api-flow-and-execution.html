Backend API and Worker Execution

Safe API Request Validate input and user access
Queued Job Save durable state and return quickly
Worker Claim job, stage files, update progress
Proxmox Action Clone, configure, start, and record
Template builds Django saves the request as a job; the worker later stages ISOs, generates Packer files, and runs the build.
VM provisioning The backend validates the selected template, asks Proxmox to create the VM, and records the result.
