PostgreSQL: The Source of Truth

The web app and the Packer worker coordinate through Django models, so durable state stays in one place while long-running build files live on shared storage.

  • Compose and deployment use PostgreSQL through DATABASE_URL; local development can fall back to SQLite.
  • Django stores AD-synced users, directory roles, saved ISO/software sources, templates, build jobs, and provisioned VMs.
  • The worker claims queued jobs with database transactions, updates stages and heartbeats, and records final results.
  • Filesystem paths, logs, staged ISOs, and archives are referenced by database rows instead of becoming the system of record.
